FAQ

Pertanyaan yang sering bikin user ragu.

Bagian ini membantu user baru merasa aman sebelum checkout dan mengurangi chat berulang ke admin.

Berapa lama proses top up?

Untuk front-end, tampilan sudah menyiapkan alur 1–3 menit. Proses real tetap perlu backend, payment gateway, dan supplier API.

Apakah total bayar sudah termasuk admin?

Ya. Ringkasan pesanan menghitung harga produk, biaya admin metode pembayaran, dan total akhir secara realtime sebelum order dibuat.

Bagaimana kalau salah User ID?

User harus mengecek ulang User ID dan server sebelum membuat order. Catatan keamanan ditampilkan di checkout dan invoice supaya kesalahan input berkurang.

Bagaimana jika sudah bayar tapi belum masuk?

Desain status order sudah disiapkan dengan timeline. Untuk kasus real, hubungkan status ke backend dan tampilkan kontak CS resmi.

Apakah bisa refund?

Area kebijakan refund sudah tersedia. Isi finalnya perlu disesuaikan dengan aturan toko dan supplier kamu.

Apakah kode order bisa dicek lagi?

Bisa, selama order dibuat dari perangkat/browser yang sama karena versi front-end ini menyimpan demo order di localStorage.
